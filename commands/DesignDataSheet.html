Design Test Data Sheet

- The-Octopus-Test framework is designed based on the DataDriven approach.

- In order to have the DataDriven feature, You need to have a test method that is designed to accept defferent test data sets.

- Each test method / application page should have a separated data sheet.

- You can follow the relational data design like SQL by refrence a row from data sheet to another.


Steps to Design a Test Data Sheet :


  • Go to the folder resources / TestData .
  • Open the file ControlFile.xlsx.
  • Follow Excel sheet how to create a new sheet.
  • Open the new created Sheet.
  • Design the new sheet as following.

    • - first column is the RowID that holds numbers of test case data rows .

    - second column is the LoginRow to refrence the login data row from the login data sheet .

    - third column is the TestCaseDescription to set a discription about the test case executed by this data row .

    - fourth column is the ExpectedResult to give an exepected result for that data row to use in the code to assert results .

    - rest of the sheet is column names like the fields on the application page/screen .

  • Follow the ControlFile.xlsx in the The-Octopus-Sample-Test-Project
