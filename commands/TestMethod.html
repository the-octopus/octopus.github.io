Test methods

The-Octopus-Test framework is designed based on the DataDriven approach.

In order to have the DataDriven feature, You need to have a test method that is designed to accept defferent test data sets.

Some said words about test method design is Do not re implement the bussiness in the test methods, Just test it.


Test methods guidlines:


- It is a mandatory to pass a dictionary to the test method.

- It is a MUST to use the dict() data collection, becuase it is a Key/Value data collection type.

- Test method is a simple blind method that only interact with the application elements by Click(), sendKeys(), ..etc.

- Test method can contains some reporting steps using the Reporter class.

- It is recommended to return a values to explain the status of the test method.

- For example, to create a test method, that is supporting DataDriven approach: 

      

        # Test Method is taking parameter of data type dictionary

        # Test method is a blind method wich is taking input data , then input data to the screen

        def login(self,tdr=dict()):

            self.set_userName(tdr["userID"])

            self.set_password(tdr["Password"])

            self.click_submit()

            return True
