Passing Data to Test methods

The-Octopus-Test framework is designed based on the DataDriven approach.

In order to have the DataDriven feature, You need to have a test method that is designed to accept defferent test data sets.


Test methods guidlines:


- It is a mandatory to pass a dictionary to the test method.

- It is a MUST to use the dict() data collection, becuase it is a Key/Value data collection type.


Passing test data to test method :

- as I explianed it is a mandatory to pass a dictionary dict() data collection to the test method.

- Why we are using the dict() data collection, becuase we need to access the test data by key/value approach.

- The data source for test data is the ControlFile.xlsx.

- Each application page/screen should have a separated data sheet in the **ControlFile.xlsx** file.

- The strucure of the Data sheet is like SQL data tables, first row should have the column names.

- It is mandatory to add a RowID column in the first column in the data sheet.

- It is mandatory to have a meaningful column names as per the fields on the application page/screen.

- Sample Code. 

       
        def login(self,tdr=dict()):

        self.set_userName(tdr["userID"])
