Test Scenario Methods

- The-Octopus-Test framework is following the KeyWord Driven approach.

- The KeyWords in the framework are the name of the scenario method in the testScenarios.py class.

- The testScenarios.py class, is mandatory to annotate all methods that will be executed as @staticmethod.

- The testScenarios.py class, is mandatory to be only one class under the path test/scenarios.

- The mapping between keywords "method name" and the test data is implement in the Driver data sheet in ControlFile.xlsx.

Steps to create a Test Scenario Method :

  • Go to the folder test / scenarios > .
  • Open the file testScenarios.py.
  • Follow the example in the The-Octopus-Sample-Test-Project, as bellow code sample.

      
        # import the octopus supportive classes
        # to report a status in the except: block
        from Octopus.OctopusHTMLReport.OctopusReporter import Reporter
        
        #inmport the test method / page class , for exmaple loginPage
        from test.pages.loginPage import LoginPage
  
        class TestScenarios:          
            @staticmethod
            def ValidateLogin(TestDataRow=dict()):
                try:
                  login_page = LoginPage(Globals.Test.Browser) # initailize object from the class loginPAge by passing the Browser.
                  login_page.login(TestDataRow) # passing the test data that is initialized in framework as per the driver data sheet configurations.
                  login_page.login_error_displayed()
                except Exception as error:            
                    Reporter.error("Validate Login Failed for {0}".format(error)) #ValidateLogin
