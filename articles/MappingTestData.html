Mapping Test Data to Scenario methods

The-Octopus-Test framework is designed based on the DataDriven approach.

In order to have the DataDriven feature, You need to have a scenario method that is designed to accept defferent test data sets.


Steps to map a Test Data to a Scenario Method:


  • Go to the folder test / scenarios .
  • Open the file testScenarios.py.
  • Copy the test scenario method name.
  • Go to the folder resources / TestData .
  • Open the file ControlFile.xlsx.
  • Open the Driver Sheet.
  • In a new row put the Name of Scenario Method in the column Function_Name.
  • In the same row put the Name of Data Sheet in the column TestDataSheetName.
  • In the same row put the row numbers of test data sheet in the column TestDataSheetRowNo.
  • The data sheet row Numbers, can be range like 1-10 , or can be list like 1,2,3,5,8
  • Follow the ControlFile.xlsx in the The-Octopus-Sample-Test-Project
