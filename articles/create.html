Test project structure

The-Octopus-Test framework is designed to be easy to use and configurable. and give more control on the execution of auotmated test cases.

In order to have these feature, You need to have a project structure that enable and help use these features.

To create a new new test project, and for a quick start I would recommend to clone git clone https://github.com/the-octopus/octopus-sample-test.git the octopus sample test project The-Octopus-Sample-Test-Project.


Folder structure

It is a mandatory to keep the naming and structure of folder as explained bellow.

For example, to create a test project at the latest you have to follow that structure: 

      
        Octopus_Sample_Test
          +--------------------------+          
          |                          |
          |reports                   |
          |                          |

Reports folder :

By default, It is auto created after test run finish.

It contains a subfolders with all exetcutions and to keep history for each run, it creates a new folder with name contains a time stamp.

Sub-folder name like Automation_Result_20201028164110.

The sub-folder contains a screen shoots subfolder with name ScreenShots_projectName and the auto generated OctopusHTMLReport with name Automation_Report_projectName.html.


Resources folder :

It contains the test browsers extensions and the test data excel sheet as well.

Sub folders are:

  • ChromeDriver

    • - it contains the Chrome Driver extension exe file.

  • FireFoxriver

    • - it contains the FireFox Driver extension exe file.

  • IEdgeDriver

    • - it contains the Edge Driver extension exe file.

  • TestData

    • - it contains the test data excel sheet ControlFile.xlsx .


Test folder :

It hold all the testing related classes for individual pages and the test scenarios class as well.

It contains 2 sub folders pages & scenarios.

  • pages

    • - we are following the POM architicture pattern, so this folder contains a python module / file for each page from the application under test.

  • scenarios

    • - It Contains only 1 module/file with the name testScenarios.py , and this class is the one that holds the test scenario methods that will be called by the driver module, these methods can combine more than one test method from different modules under the pages class.


main .py file:

This module is the main entry point for the execution.

In order to run our project from the command line as follow.

- open command line.

- cd to the folder contains the main .py.

- run the command

       
       python main.py
